## 三、主要创新点

1. **唯一核心对象 `SkillPolicy`**  
   统一策略结构、状态条件、健康指标、置信度、错误记忆、**RL权重(θ)** 及 **稳定性评分**。

2. **策略分布学习（Policy Gradient）**  
   可学习的策略分布 `π(p|s) = softmax(semantic + regime + θ_bias)`，θ 通过 Policy Gradient 在线更新，实现策略偏好的自适应演化。

3. **三大稳定性 Patch（数值稳定核心）**
   - **Patch 1（Evolution Soft Gate）**：`stability_score = 0.6*EMA + 0.4*(1-error_volatility)`，不稳定策略更易被扰动。
   - **Patch 2（Adaptive Decay）**：`decay = base_decay / sqrt(selection_count + 1)`，高频策略保持记忆，低频策略加速遗忘。
   - **Patch 3（θ Bias Injection）**：`theta_norm = tanh(θ)`，θ 归一化后作为 logit bias 注入，防止 RL 支配 softmax。

4. **预测缓存复用（⚡ 提速核心）**  
   全局缓存 `rl_cache.pkl` 存储所有策略在 B 子集窗口的预测结果，RL 训练时直接读取，**节省 95% 以上时间**。

5. **新策略试用期冻结**  
   新 `TRIAL` 策略在前 2 轮仅能被探索选中且不更新 θ，积累足够数据后再参与竞争，避免“新手”冲击 Baseline。

6. **分策略学习率**  
   `TRIAL` 策略使用高学习率（快速适应），`ACTIVE` 策略使用低学习率（保护已训练参数），实现“快慢结合”。

7. **质量门控（过滤低质量补丁）**  
   Re-Induction 候选策略若 MASE 不低于池中最差策略，则被过滤，减少无效扰动和池子膨胀。

8. **概率补偿（抗稀释）**  
   新增策略时，对旧策略 θ 加上 `log((N+K)/N)`，保持旧策略采样概率不变，实现平滑过渡。

9. **测试时 Top‑2 集成投票**  
   在 Ablation 评估中，取适用性最高的前 2 个策略的预测加权平均，降低测试 MASE。

10. **动态学习率自适应**  
    根据每轮 MASE 变化趋势调整学习率：性能提升时上调，性能下降时下调。

## 四、每一轮的执行步骤及作用

### 第 1 轮：策略归纳（A部分）

| 步骤 | 操作 | 作用 |
|------|------|------|
| 1.1 | 加载 A 部分窗口（152个） | 数据准备 |
| 1.2 | 逐窗口 LLM 生成 2 个候选策略 | 利用 LLM 创造力生成多样化策略 |
| 1.3 | 技能有效性校验（过滤无效技能） | 确保策略可执行 |
| 1.4 | 策略聚类（Policy Space Partitioning） | 合并同类策略，控制池子规模 |
| 1.5 | 生成初始 `SkillPolicy` 对象 | 统一数据结构 |
| 1.6 | 初始化 θ=0，加入分布模型 | 所有策略地位平等 |
| 1.7 | 保存策略和检查点 | 持久化，支持断点续训 |

**目标**：生成覆盖不同场景的初始策略池，所有策略 θ=0。

## 二、参数的设计哲学与核心价值

### 2.1 为什么需要这个参数？

在 SPLS 系统中，每个策略本质上是一个“静态的知识片段”（包含条件 + 执行策略）。问题在于：

- **第 1 轮归纳**生成的策略是**静态的**，无法适应数据分布的变化。
- 不同策略在不同数据状态下表现各异，但系统不知道哪个策略“更靠谱”。
- 需要一个**轻量级的可学习信号**，在不修改策略结构的前提下，动态调整策略的“受青睐程度”。

### 2.2 设计来源与创新

θ 的设计灵感来自**强化学习中的 Policy Gradient 方法（如 REINFORCE 算法）**，但 SPLS 的创新在于：

| 常规 RL | SPLS-RL |
|---------|---------|
| 学习一个**大型神经网络**的权重 | 只学习一个**轻量级偏置向量**（每个策略一个标量） |
| 策略是**隐式**的（由网络参数定义） | 策略是**显式**的（由 LLM 生成的规则定义），RL 只调整“选择偏好” |
| 需要大量样本训练 | 每个窗口即是一个样本，在线更新 |

**SPLS 的创新点**：将 RL 的“探索-利用”机制与 LLM 生成的显式策略池结合，用极低的成本实现了策略池的“自净化”和“自适应”。

### 2.3 θ 的核心价值

| 价值维度 | 说明 |
|----------|------|
| **动态适应** | 当数据分布变化时，θ 自动将概率从旧策略转移到新策略，无需重新训练 |
| **探索驱动** | 没有 θ 的系统只会“利用”（选语义最匹配的），有 θ 的系统会“探索”（尝试潜在好策略） |
| **数据积累** | θ 驱动的高频采样为策略积累了统计显著的 `error_history` 和 `avg_mase` |
| **策略净化** | θ 驱动的冷落最终导致差策略被标记为 `DEPRECATED`，实现策略池的优胜劣汰 |

## 四、训练时 θ 的作用（v6.1 增强版）

### 4.1 作为 Logit 偏置，参与采样概率计算

在每轮 RL 训练中，系统计算每个策略被选中的概率：

```
logit = semantic_score + α * tanh(θ) + regime_bonus          ← Patch 3 + Regime偏置
P(policy) = softmax(logit)
```

| 组成部分 | 含义 | 特点 |
|----------|------|------|
| `semantic_score` | 策略“适合做什么”（由 `embedding` 和 `state_condition` 决定） | **静态的**，策略的固有属性 |
| `tanh(θ)` | θ 的归一化值，范围 `[-1, 1]` | **动态的**，反映策略“最近表现如何” |
| `α` | θ 的贡献权重（`theta_bias_alpha = 0.15~0.20`） | 控制 RL 对决策的影响程度 |
| `regime_bonus` | 基于策略在历史 Regime 下的表现奖励 | 提高“状态-策略”匹配精准度 |

### 4.2 驱动“探索与利用”的平衡（含试用期冻结）

| 行为 | 触发条件 | 效果 |
|------|----------|------|
| **探索（Exploration）** | ε-greedy + UCB 选择 | 可选中冻结期 TRIAL 策略，探索未知潜力 |
| **利用（Exploitation）** | 按分布采样，但排除冻结期策略 | 稳定利用已验证的好策略，避免新手干扰 |
| **惩罚（Punishment）** | `θ < 0` 的策略被低概率选中 | 表现差的策略被冷落，逐渐被淘汰 |
| **冻结期保护** | TRIAL 策略在 2 轮内不更新 θ | 积累足够数据再参与竞争，降低“新手运气”扰动 |

### 4.3 影响策略池的统计指标

每次策略被执行后，系统会更新该策略的以下字段：

| 字段 | 更新方式 | 用途 |
|------|----------|------|
| `error_history` | 追加新的误差值 | 计算 `avg_mase`、`error_volatility` |
| `avg_mase` | 滑动平均 | 反映策略的真实预测能力 |
| `reward_ema` | 指数移动平均 | 反映策略的长期效用 |
| `utility_ema` | `reward_ema` 的映射 | **退休决策的核心依据** |
| `selection_count` | 累加 | 统计策略被采样的次数（影响 decay 和冻结期判断） |
| `regime_performance` | 按 Regime 分组记录 MASE | 用于计算 `regime_bonus`，提升匹配精度 |


## 五、测试时 θ 的作用（完全弃用）

### 5.1 测试时的策略选择逻辑

在最终测试（Ablation）中，策略选择的唯一依据是：

```
best_policy = argmax(compute_applicability_score(state))
```

其中：
```
compute_applicability_score = 0.7 * cos_sim(embedding, state_vector) + 0.3 * hard_condition_match
```

**这个公式里没有任何 θ 或 logit_weight 的参与。**

### 5.2 为什么测试时不用 θ？

| 原因 | 说明 |
|------|------|
| **防止过拟合** | θ 是在特定 B 子集（训练窗口）上训出来的，直接用于测试集会过度偏向训练分布 |
| **保持纯净推理** | θ 反映的是“历史上谁更受偏爱”，而不是“当前状态谁更合适” |
| **避免加权拖累** | 用 θ 做软混合会拖累最佳策略的预测精度 |

### 5.3 θ 的“间接影响”

虽然测试时 θ 被弃用，但它通过训练过程**间接改变了策略池的构成**：

| θ 驱动的变化 | 对测试的影响 |
|-------------|-------------|
| 差策略被标记为 `DEPRECATED` | 测试时候选池更纯净，避免灾难性策略被误选 |
| 好策略积累了可靠的 `avg_mase` | 策略的健康指标可信，为后续维护提供依据 |
| 补丁策略（`reind_`）被生成 | 困难窗口有专门的“特种兵”策略应对 |
| 质量门控确保只有益策略加入 | 测试集上的平均性能提升 |

## 七、三个 Patch 对 θ 的稳定化保障（v6.1 增强版）

### 7.1 Patch 1：Evolution Soft Gate（稳定策略扰动）

```
stability_score = 0.6 * reward_ema + 0.4 * (1 - error_volatility)
p_evo = strength * (1 - stability_score)    ← 限幅 [0.05, 0.4]

if random() < p_evo:
    执行演化动作（合并/退休）
else:
    仅降权（utility_ema *= 0.95）
```

**核心思想**：时间尺度分离——`reward_ema` 是长期信号，`error_volatility` 是短期信号。不稳定策略（`stability_score` 低）被扰动的概率更高。

### 7.2 Patch 2：Adaptive Decay + 概率补偿（防止 θ 爆炸 + 抗稀释）

```
adaptive_decay = base_decay / sqrt(selection_count + 1)
θ = (1 - adaptive_decay) * θ + lr * Advantage

★ 新增：当新策略加入时，对旧策略执行概率补偿
θ_old_compensated = θ_old + log((N+K)/N)
```

**核心思想**：
- 高频访问策略 → 更小 decay（保持记忆）
- 低频策略 → 更大 decay（加速遗忘）
- 防止 θ 无限增长（softmax collapse）
- **概率补偿** 确保旧策略的采样概率不被新策略稀释

### 7.3 Patch 3：θ Bias Injection + 裁剪（防止 RL 支配语义）

```
theta_norm = tanh(θ)                        ← 缩放到 [-1, 1]
logit = semantic_score + alpha * theta_norm + regime_bonus

Reward = max(-MASE, -10.0)                  ← Reward 裁剪
Advantage = clip(Reward - Baseline, -10, 10) ← Advantage 裁剪
```

**核心思想**：
- θ 范围 `[-∞, +∞]`，语义分数 `[0, 1]`。如果不归一化，θ 会支配 softmax，导致 collapse。
- `tanh` 归一化后，θ 作为**归一化 logit bias** 参与决策。
- **双重裁剪** 消除极端 MASE 对 θ 更新的冲击。

### 7.4 额外稳定措施：试用期冻结 + 分策略学习率

| 措施 | 公式/逻辑 | 作用 |
|------|----------|------|
| **试用期冻结** | TRIAL 策略在前 2 轮不更新 θ | 避免“新手运气”冲击 Baseline |
| **分策略学习率** | TRIAL: lr×2.0, ACTIVE: lr×0.5 | 快慢结合，保护已训参数 |

## 九、与 `embedding` 的关键区别（无变化）

| 维度 | **`embedding`（策略嵌入）** | **`logit_weight`（θ 偏置）** |
|------|----------------------------|------------------------------|
| **来源** | 随机初始化 → 聚类质心 → Refresh 漂移修正 | 初始化为 0 → Policy Gradient 更新 |
| **本质** | **静态身份**（策略“适合解决什么问题”） | **动态声誉**（策略“最近解决得怎么样”） |
| **是否被梯度更新** | **否**（仅通过 Refresh 规则调整） | **是**（通过 RL 的 Advantage 信号更新） |
| **训练时的作用** | 作为基础分数（`semantic_score`） | 作为可学习偏置（`tanh(θ)`） |
| **测试时是否使用** | **是**（核心匹配依据） | **否**（完全弃用） |
| **最终影响** | 影响策略被匹配的**空间位置** | 影响策略在训练时被**探索和试错**的机会 |


## 十、最终本质（v6.1 升华）

> **`logit_weight` 是系统在持续变化的数据流中，用于动态调节“探索与利用”平衡的旋钮。它通过影响采样频率来驱动策略池的自我净化，但它本身从不直接参与最终的测试决策——它的全部意义，在于为最终被选中的那个“精锐策略”的诞生，提供足够的“试错机会”和“数据养分”。**  
> **★ v6.1 增强**：θ 的更新现在受到多重保护机制（裁剪、冻结、分学习率、概率补偿），使其在策略池动态变化时依然能稳定收敛，避免了早期版本中因新策略加入导致的剧烈波动。

### 补充：θ 的核心价值（v6.1 扩展）

| 价值维度 | 说明 |
|----------|------|
| **动态适应** | 当数据分布变化时，θ 自动将概率从旧策略转移到新策略，无需重新训练 |
| **探索驱动** | 没有 θ 的系统只会“利用”（选语义最匹配的），有 θ 的系统会“探索”（尝试潜在好策略） |
| **数据积累** | θ 驱动的高频采样为策略积累了统计显著的 `error_history` 和 `avg_mase` |
| **策略净化** | θ 驱动的冷落最终导致差策略被标记为 `DEPRECATED`，实现策略池的优胜劣汰 |
| **稳定演化**（新） | 通过试用期冻结、概率补偿、分策略学习率等机制，θ 在策略池动态变化时保持平稳收敛 |
| **质量保障**（新） | 质量门控确保只有真正有益的补丁才能加入，减少 θ 的无效扰动 |
