## 2. 添加新数据源（如股票、电力）或修改问题模板

所有数据源和问题模板都集中在 **`data/dataset_registry.yaml`** 文件中管理。你只需修改该文件，并确保对应的数据文件（Parquet/CSV）或 API 实现存在即可。

### 2.1 添加静态文件数据源（例如股票）

以添加“贵州茅台股票每日收盘价”为例，假设你已经有一个 `data/stock/maotai.parquet` 文件，包含 `date` 和 `close` 列。

**① 编辑 `data/dataset_registry.yaml`**，在 `datasets` 列表末尾添加：

```yaml
  - id: stock_maotai
    name: "贵州茅台每日收盘价"
    domain: "stock"
    frequency: "daily"
    mode: "static"                     # 静态文件模式
    source:
      type: "file"
      class: "FileDataSource"
      params:
        path: "data/stock/maotai.parquet"
    target_column: "close"
    resolution_delay_days: 1
    volatility_window: 30
    templates:
      - id: price_next_day
        type: "numerical"
        question_template: "预测贵州茅台明天({date})的收盘价"
        variables: ["date"]
        horizon: 1
        answer_extraction: "从文件读取{date}的close"
        difficulty: 2
      - id: price_5day_avg
        type: "numerical"
        question_template: "预测贵州茅台未来{horizon}天的平均收盘价"
        variables: ["horizon"]
        horizon_options: [3,5,10]
        answer_extraction: "读取[date, date+horizon)的close平均值"
        difficulty: 3
```

**② 将数据文件放入对应目录**，确保 Parquet 文件包含 `date`（日期列）和 `close`（目标列），日期已排序。

**③ 无需修改任何 Python 代码**。`TaskGenerator` 会自动读取 registry，使用 `dataset/loader.py` 加载文件，`splitter.py` 切分历史/未来。答案收集器 `AnswerCollector` 中的逻辑需要略微扩展以支持通用的 `answer_extraction` 描述，当前示例中硬编码了 temp 和 demand 的处理，你可以修改 `src/evaluation/collector.py` 使其更通用（见下文扩展建议）。

### 2.2 添加 API 数据源（不付费、易获取，见第3节）

若使用免费 API（如 yfinance 获取股票数据），需要在 `src/data_fetcher/` 下实现对应的 `DataSource` 子类，并修改 `dataset_registry.yaml` 中的 `source.type` 为 `"api"`，`class` 为你实现的类名。

### 2.3 修改已有问题模板

直接编辑 `data/dataset_registry.yaml` 中对应数据集的 `templates` 列表即可。你可以：
- 修改 `question_template` 的中文描述
- 调整 `horizon_options` 改变可选的预测步长
- 改变 `difficulty` 分数
- 增加更多变量（需同步修改 `TaskGenerator._fill_variables` 方法）

## 3. 设置免费易获取的数据源

以下是两种完全免费、无需付费 API 的实现方案：

### 方案 A：使用 yfinance 自动拉取股票数据（免费，无 Key）

`yfinance` 可以下载雅虎财经的历史股票数据，无需注册和 API Key。

**① 安装依赖**：在 `requirements.txt` 中添加 `yfinance>=0.2.0`，并 `pip install yfinance`。

**② 实现 DataSource 子类**：在 `src/data_fetcher/stock.py` 中（或新建 `src/data_fetcher/yfinance_source.py`）：

```python
import pandas as pd
import yfinance as yf
from .base import DataSource

class YFinanceDataSource(DataSource):
    def fetch(self, symbol, start="2020-01-01", end=None, **kwargs):
        if end is None:
            end = pd.Timestamp.today().strftime("%Y-%m-%d")
        df = yf.download(symbol, start=start, end=end, progress=False)
        df = df.reset_index()
        df = df.rename(columns={"Date": "date", "Close": "close"})[["date", "close"]]
        df["date"] = pd.to_datetime(df["date"])
        return df.sort_values("date")
```

**③ 在 `data_fetcher/base.py` 中**确保 DataSource 抽象类存在：

```python
from abc import ABC, abstractmethod

class DataSource(ABC):
    @abstractmethod
    def fetch(self, *args, **kwargs):
        pass
```

**④ 配置 registry 使用该 API**：

```yaml
  - id: stock_maotai
    name: "贵州茅台每日收盘价"
    domain: "stock"
    frequency: "daily"
    mode: "realtime"                     # 实时拉取
    source:
      type: "api"
      class: "YFinanceDataSource"
      params:
        symbol: "600519.SS"             # 茅台代码
        start: "2023-01-01"
    target_column: "close"
    resolution_delay_days: 1
    volatility_window: 30
    templates:
      - id: price_next_day
        type: "numerical"
        question_template: "预测贵州茅台明天({date})的收盘价"
        variables: ["date"]
        horizon: 1
        answer_extraction: "从API读取{date}的close"
        difficulty: 2
```

**⑤ 修改 `dataset/loader.py`**，使其能调用 API：

```python
from src.data_fetcher.stock import YFinanceDataSource  # 或动态导入

def load_dataset(dataset_config):
    source = dataset_config['source']
    if source['type'] == 'file':
        path = source['params']['path']
        df = pd.read_parquet(path) if path.endswith('.parquet') else pd.read_csv(path)
        df['date'] = pd.to_datetime(df['date'])
        return df.set_index('date').sort_index()
    elif source['type'] == 'api':
        class_name = source['class']
        # 简单映射，可扩展为动态导入
        if class_name == 'YFinanceDataSource':
            ds = YFinanceDataSource()
        else:
            raise ValueError(f"Unknown API class: {class_name}")
        df = ds.fetch(**source['params'])
        return df.set_index('date').sort_index()
```

**⑥ 答案收集器**也需要适配 API 模式，可以使用同样的 API 重新拉取对应日期数据来获取真实值。

### 方案 B：使用公开静态数据集（CSV/Parquet，完全离线）

许多平台提供免费的时序数据集，你只需下载后放置到 `data/` 目录，然后在 registry 中注册为 `static` 文件类型即可。例如：
- **Electricity Load Diagrams**（UCI）
- **METR-LA 交通数据**
- **天气数据**：可以从 NOAA 下载历史天气 CSV。

无需编写任何 Python 代码，只改 YAML 配置即可。这是最简单的零成本方式。

### 推荐组合

- **股票数据**：yfinance 免费且易用，适合动态更新。
- **电力、天气等**：使用预下载的 Parquet 文件，避免依赖外部 API。
- **问题模板修改**：仅编辑 `dataset_registry.yaml`。

通过以上方法，你可以在几分钟内增加任意数量的免费数据源，并自定义预测问题。

### 2. `src/tasks/generator.py`
修改任务生成逻辑，支持通过 `start_override` / `end_override` 覆盖配置里的时间，并把每个加载失败的数据源详情收集到 `failed_collector` 列表中。

```python
import random
import uuid
import pandas as pd
from datetime import datetime, timedelta
from src.tasks.instance import TaskInstance
from src.dataset.registry import DatasetRegistry
from src.dataset.loader import load_dataset
from src.dataset.splitter import split_history_future
from src.config import TASKS_DIR
import os

class TaskGenerator:
    def __init__(self):
        self.registry = DatasetRegistry()
        self.datasets = self.registry.datasets

    def generate_daily_tasks(self, date_str=None, num_tasks=10,
                             start_override=None, end_override=None,
                             failed_collector=None):
        """
        生成每日任务。

        Args:
            date_str: 任务日期。
            num_tasks: 目标任务数。
            start_override: 覆盖所有数据源的起始日期。
            end_override: 覆盖所有数据源的结束日期。
            failed_collector: 可选列表，用于收集不可用数据源的信息。
        Returns:
            成功生成的 TaskInstance 列表。
        """
        if date_str is None:
            date_str = datetime.now().strftime("%Y-%m-%d")
        gen_date = datetime.strptime(date_str, "%Y-%m-%d")
        tasks = []
        dataset_list = list(self.datasets.values())
        random.shuffle(dataset_list)

        attempts = 0
        max_attempts = num_tasks * 3  # 防止死循环
        failed_sources = {}  # 临时记录失败的数据源，避免重复打印

        while len(tasks) < num_tasks and attempts < max_attempts:
            attempts += 1
            ds = random.choice(dataset_list)
            template = random.choice(ds['templates'])
            variables = self._fill_variables(template, gen_date)

            # ----- 覆盖起止日期 -----
            ds_config = ds.copy()
            if start_override or end_override:
                ds_config['source']['params'] = ds_config['source']['params'].copy()
                if start_override:
                    ds_config['source']['params']['start'] = start_override
                if end_override:
                    ds_config['source']['params']['end'] = end_override
            # -------------------------

            try:
                df = load_dataset(ds_config)
                history = split_history_future(df, ds['target_column'], variables, template)
            except Exception as e:
                # 记录失败信息（每个数据集只记录一次）
                ds_id = ds['id']
                if ds_id not in failed_sources:
                    failed_sources[ds_id] = {
                        'id': ds_id,
                        'name': ds.get('name', ds_id),
                        'error': str(e)
                    }
                continue

            horizon = variables.get('horizon', template.get('horizon', 1))
            resolution_date = gen_date + timedelta(days=horizon + ds['resolution_delay_days'])
            task = TaskInstance(
                id=f"{ds['id']}_{date_str}_{uuid.uuid4().hex[:8]}",
                dataset_id=ds['id'],
                template_id=template['id'],
                question=template['question_template'].format(**variables),
                question_type=template['type'],
                history=history,
                horizon=horizon,
                prediction_target=variables,
                resolution_date=resolution_date,
                difficulty_level=template.get('difficulty', 1),
                ground_truth_extractor=template['answer_extraction']
            )
            tasks.append(task)

        # 将失败的数据源信息写回传入的列表（供调用方展示）
        if failed_collector is not None:
            failed_collector.extend(failed_sources.values())

        os.makedirs(TASKS_DIR, exist_ok=True)
        task_path = os.path.join(TASKS_DIR, f"{date_str}.jsonl")
        with open(task_path, 'w', encoding='utf-8') as f:
            for task in tasks:
                f.write(task.model_dump_json() + "\n")

        if len(tasks) < num_tasks:
            print(f"⚠️ 只成功生成 {len(tasks)}/{num_tasks} 个任务（部分数据源不可用）")
        return tasks

    def _fill_variables(self, template, gen_date):
        vars = {}
        if 'variables' in template:
            for v in template['variables']:
                if v == 'date':
                    vars['date'] = (gen_date + timedelta(days=random.randint(1, 7))).strftime("%Y-%m-%d")
                elif v == 'horizon':
                    opts = template.get('horizon_options', [1, 3, 7])
                    vars['horizon'] = random.choice(opts)
        return vars
```

```bash
python run.py --date 2024-06-01 --num_tasks 5 --model glm-4-flash
```

### 2) 只覆盖起始日期
```bash
python run.py --date 2024-06-01 --num_tasks 5 --start 2023-01-01
```

### 3) 同时覆盖起止日期
```bash
python run.py --date 2024-06-01 --num_tasks 5 --start 2022-01-01 --end 2024-12-31
```

### 4) 强制更小的数据范围（例如仅使用最近 3 个月的数据训练/预测）
```bash
python run.py --date 2024-06-01 --num_tasks 5 --start 2024-03-01 --end 2024-06-01
```
