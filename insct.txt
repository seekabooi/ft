升级后的评估方案：滚动起源验证 (Rolling Origin)
这是时间序列预测中最严谨的评估方式之一，能最大化利用数据，同时保证无未来信息泄露。

核心思想：
设置一个最小训练窗口（如 50 个数据点），然后从该窗口末尾开始，反复进行“训练→预测→评估→窗口前移”的循环，直到用完所有数据。
数据总长 N = 144 (airline)
min_train_len = 80   (至少用80个点训练)
horizon = 1

Step 0: 训练窗口 [0:80]，预测第81个点，记录误差
Step 1: 训练窗口 [1:81]，预测第82个点，记录误差
...
Step 64: 训练窗口 [63:143]，预测第144个点，记录误差

共得到 N - min_train_len = 64 次预测，计算平均 RMSE/MAE
优点：数据利用率 100%，结果更稳定，且能看出模型在不同时期的性能变化。

### 一、详细的判断处理流程图

```
┌─────────────────────────────────────────────────────────────────┐
│                      开始：滚动起源评估                            │
│   for each test window (min_train_size 后滑动)                   │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 步骤1：多尺度特征提取 (DataProfiler)                              │
│  - 自动周期检测 (ACF)                                            │
│  - 统计特征：ADF p值、季节性强度、趋势强度、数据长度、波动率         │
│  - 自适应窗口栈：长(3*period)、中(1*period)、短(max(2,period/4))、 │
│    差分(20) → 归一化 → 拼接 snapshot_vector                       │
│  - 对极度非平稳(ADF>0.9)强制季节性强度归零                          │
│  - 输出 profile                                                  │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 步骤2：技能匹配 (SkillMatcher)                                    │
│   阶段1（硬过滤）：遍历所有已注册技能                               │
│     - skill.check_state_card(profile)                             │
│     - 必须满足 when_to_use 且不触发 when_not_to_use                │
│     - 未通过直接淘汰                                              │
│   阶段2（软排序）：对通过技能                                       │
│     - 计算当前 snapshot_vector 与技能原型向量的 DTW 相似度 (若存在)  │
│     - 按相似度降序，取 top_k (默认3)                               │
│   输出：候选技能列表                                               │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 步骤3：LLM 分支决策 (_llm_branch_decision)                        │
│  - 将 profile 和候选技能的 state_card 发送给 LLM                   │
│  - 可选：加入候选技能在最近局部窗口的回测误差(MAE)                   │
│  - LLM 返回 JSON 格式的指导：                                      │
│    {                                                              │
│      "best_skill": "arima",          ← 单个技能                   │
│      或 "ensemble_skills": ["arima","prophet"] ← 动态组合         │
│    }                                                              │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 步骤4：执行预测 (Agent.predict)                                    │
│  - 若 guidance 中有 ensemble_skills：                             │
│     动态创建 EnsembleSkill → 执行各技能 → 返回均值                  │
│  - 若 guidance 中有 best_skill：                                  │
│     调用 skill.execute(history, horizon, period=period)            │
│  - 使用 skill.verify_prediction() 验证                            │
│  - 若验证失败，使用候选中的 fallback_skill                         │
│  - 所有技能失败时，回退到 naive 或 ensemble(all)                    │
│  - 最终保底：历史均值（极少触发）                                    │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 步骤5：滚动窗口前进 (步长 = horizon)                               │
│  - 将当前测试点的真实值加入历史                                    │
│  - 重复步骤1-4，直到测试集结束                                      │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 步骤6：指标计算 (RollingOriginEvaluator)                           │
│  - 计算 RMSE, MAE, sMAPE (标准公式), MASE (基于全局训练尺度)        │
│  - 进度条显示，日志记录所有决策                                    │
└─────────────────────────────────────────────────────────────────┘
```

### 三、Skill 的使用逻辑

1. **技能注册**  
   所有技能通过 `SkillRegistry` 注册，每个技能包含可执行的 `execute()` 方法和描述适用条件的 `state_card`。

2. **状态感知**  
   `DataProfiler` 提取当前时间窗口的统计特征（平稳性、季节性强度、趋势强度、周期等）和多尺度窗口快照。

3. **硬过滤**  
   `SkillMatcher` 检查每个技能的 `state_card`，仅保留满足 `when_to_use` 且不触发 `when_not_to_use` 的技能。

4. **软排序**  
   对通过过滤的技能，计算其原型向量（历史成功案例的快照）与当前窗口的 DTW 相似度，按相似度排序。

5. **LLM 决策**  
   候选技能及其状态卡、局部回测误差（如果可用）被发送给 LLM，LLM 返回 JSON，指定使用单个技能或动态组合多个技能。

6. **执行预测**  
   Agent 根据 LLM 返回的指导调用对应技能的 `execute()` 方法，获取数值预测。若验证失败，回退到下一候选或兜底技能（naive/ensemble）。

7. **记录与进化**  
   预测成功后，当前窗口快照可被存入对应技能的原型库（`prototypes`），供后续相似度匹配使用。失败案例可被 `SkillGenerator` 收集，用于归纳新的状态卡草案（需人工确认后方可转为可执行代码）。

### 四、LLM 在本项目中的作用

LLM（当前使用智谱 `glm-4`）在整个框架中**仅扮演决策者角色，不参与数值计算**。

| 作用 | 说明 |
|------|------|
| **技能选择** | 接收 `DataProfiler` 提取的统计特征和候选技能的 `state_card` 文本，决策使用哪一个（或哪些）技能。 |
| **动态组合** | 可以在返回的 JSON 中指定 `ensemble_skills` 列表，让 Agent 临时组合多个技能取均值。 |
| **兜底预测**（仅无技能模式） | 在 `--no_skills` 纯 LLM 模式下，LLM 根据极简 Prompt 直接输出数值。 |
| **状态卡归纳**（可选） | 在 `SkillGenerator` 中，LLM 根据失败案例特征生成新技能的状态卡草案（需人工审核）。 |
| **局部误差参考**（增强） | 可接收候选技能在局部验证窗口上的回测误差，帮助做出更明智的选择。 |

**LLM 不参与**：
- 特征提取（纯 Python 计算）
- 状态卡条件匹配（纯规则判断）
- 技能执行（调用本地算法/模型）
- 预测验证（Z-score 等统计检验）
- 指标计算（RMSE/MAE/sMAPE/MASE）

- `trend_strength`（趋势强度，0~1）
- `data_length`（数据长度）
- `recent_volatility`（近期波动率）
- `period`（自动检测的周期）
- 多尺度窗口快照（`snapshot_vector`，用于原型匹配）

---

### 2. 硬过滤：`SkillMatcher` 根据状态卡筛选

所有已注册的技能都有一个 **`state_card`**（状态卡），里面定义了：

- `when_to_use`：必须同时满足的条件（如 `seasonal_strength > 0.5` 且 `data_length ≥ 2*period`）
- `when_not_to_use`：任一触发则排除该技能（如 `trend_strength > 0.6` 则不用）

`SkillMatcher` 遍历每个技能，调用 `skill.check_state_card(profile)`，**不满足条件的直接淘汰**，只保留**硬性合格**的候选技能。

---

### 3. 软排序：基于原型向量（可选）

对于通过硬过滤的技能，如果它们拥有**历史成功案例的原型向量**（存储在 `prototypes` 中），系统会计算当前窗口的 `snapshot_vector` 与这些原型向量的 DTW 相似度，按相似度从高到低排序。原型库越丰富，排序越有意义。

---

### 4. LLM 决策：最终拍板

候选技能（通常 2~3 个）及其状态卡、局部回测误差（如果可用）会被**发送给 LLM（如 `glm-4`）**，LLM 返回一个 JSON，例如：

```json
{ "best_skill": "arima" }
```

或选择动态组合：

```json
{ "ensemble_skills": ["arima", "prophet"] }
```

LLM 在此**仅做决策**，不参与任何数值计算。

---

### 5. 执行与兜底

- 若 LLM 选定了 `best_skill`，系统调用该技能的 `execute()` 进行预测。
- 预测后会通过 `verify_prediction()`（Z-score 检查）验证合理性。若验证失败，自动回退到候选列表中的下一个技能。
- 若所有候选技能均失败，最终回退到移动平均（`NaiveSkill`）或均值。

---

### 总结：决策链路

```
DataProfiler (特征提取)
    ↓
SkillMatcher (硬过滤 + 软排序)
    ↓
LLM (阅读状态卡和误差，决定 best_skill 或 ensemble)
    ↓
Agent 执行技能，验证，必要时回退
```

**LLM 是“选择者”，技能是“执行者”**。整个逻辑确保预测方法始终与当前数据状态相匹配。

307307 何时用于不用
技能“何时使用 / 何时不用”的逻辑通过三处协同实现：

状态卡定义
每个技能文件（如 src/skills/seasonal_naive.py）的 __init__ 里定义了 self.state_card，包含 when_to_use 和 when_not_to_use 的条件列表。

状态感知
src/skills/data_profiler.py 中的 DataProfiler.profile() 实时提取当前窗口的统计特征（ADF p值、季节性强度、趋势强度等），形成 profile。

匹配决策
src/skills/skill_matcher.py 中的 SkillMatcher.match() 调用 skill.check_state_card(profile)（该方法在 src/skills/base.py 中实现），逐一评估每个技能的 when_to_use 和 when_not_to_use，只让完全满足条件的技能通过，实现“何时用”与“何时不用”。

309309多尺度与历史记录
多尺度窗口和历史记录（原型向量）是 TSSkills 进行**精准状态感知**和**技能选择**的两大关键机制。

---

## 一、多尺度窗口

**实现文件**：`src/skills/data_profiler.py` 中的 `DataProfiler.profile()`。

### 1. 为什么需要多尺度窗口？

单一长度的窗口无法全面捕捉序列特征，因此系统自动构建四个互补窗口：

- **长窗口**（长度 = 3 × 周期）：捕捉长期趋势、低频季节性。
- **中窗口**（长度 = 1 × 周期）：捕捉完整周期模式。
- **短窗口**（长度 = max(2, 周期/4)）：捕捉近期波动和局部突变。
- **差分窗口**（固定 20 个点的一阶差分）：捕捉变化率。

### 2. 如何处理？

每个窗口内的数值被归一化到 [0,1] 区间，然后拼接成一个统一的 `snapshot_vector`。这个向量既作为当前状态的“数值指纹”，也用于后续原型匹配。

---

## 二、历史记录（原型向量）

**记录位置**：`src/evaluation/rolling_origin_evaluator.py` 中的 `_record_prototype()`。  
**使用位置**：`src/skills/skill_matcher.py` 中的 `SkillMatcher.match()`。

### 1. 怎么记录？

每次预测成功后，系统会将当前窗口的 `snapshot_vector` 连同误差、日期等信息，存入该技能对象的 `prototypes` 列表。默认保留最近 50 个成功案例，旧案例自动淘汰。

### 2. 怎么用？

当新窗口需要匹配技能时，对于每个通过硬过滤的技能，系统计算当前 `snapshot_vector` 与该技能所有原型向量的 DTW 距离，取最近距离转换为相似度（相似度 = 1/(1+距离)）。

- 原型相似度**高**：说明当前数据状态与该技能历史成功案例相似，更适合调用。
- 原型相似度**低**：即使状态卡允许，也可能不是最佳选择。

### 3. 作用

原型匹配实现了“基于经验的软排序”，弥补了硬阈值状态卡的不足。它让系统能够从过去的成功/失败中学习，逐渐形成“在什么情况下用什么技能最有效”的偏好。

---

**简单总结**：  
- 多尺度窗口 = 用多个“镜头”看清当前数据。  
- 历史原型 = 记住类似情况下什么技能靠谱，下次优先选用。

### 1. `src/skills/data_profiler.py`（完整版）

```python
import numpy as np
from statsmodels.tsa.stattools import adfuller, acf

class DataProfiler:
    @staticmethod
    def profile(history: np.ndarray, window_sizes: list = None, auto_period: bool = True) -> dict:
        if window_sizes is None:
            window_sizes = [len(history)]
        base = DataProfiler._statistical_profile(history, auto_period)
        snapshots = {}
        for w in window_sizes:
            if w <= len(history):
                seg = history[-w:]
                seg_min, seg_max = np.min(seg), np.max(seg)
                seg_norm = (seg - seg_min) / (seg_max - seg_min + 1e-8)
                snapshots[f'w{w}'] = seg_norm.tolist()
        if len(history) >= 2:
            diff = np.diff(history[-20:])
            d_min, d_max = np.min(diff), np.max(diff)
            if d_max > d_min:
                diff_norm = (diff - d_min) / (d_max - d_min)
            else:
                diff_norm = np.zeros_like(diff)
            snapshots['diff20'] = diff_norm.tolist()
        base['snapshots'] = snapshots
        base['snapshot_vector'] = np.concatenate([np.array(v) for v in snapshots.values()]).tolist() if snapshots else []
        return base

    @staticmethod
    def _statistical_profile(history, auto_period=True):
        n = len(history)
        if n < 10:
            return {'adf_pvalue':0.5, 'seasonal_strength':0.0, 'trend_strength':0.0,
                    'missing_rate':0.0, 'data_length':n, 'recent_volatility':0.0, 'period':7}
        try:
            adf_p = adfuller(history, maxlag=min(12, n//2))[1]
        except:
            adf_p = 0.5
        period = DataProfiler._auto_period(history) if auto_period else 7
        if n >= 2*period:
            seasonal_diff = history[period:] - history[:-period]
            seas = max(0.0, 1 - np.std(seasonal_diff) / (np.std(history)+1e-8))
        else:
            seas = 0.0
        if adf_p > 0.9:
            seas = 0.0
        diffs = np.diff(history)
        trend = abs(np.mean(diffs>0)-0.5)*2 if len(diffs)>0 else 0.0
        recent_vol = np.std(history[-5:])/np.std(history) if n>=5 and np.std(history)>0 else 1.0
        return {
            'adf_pvalue': adf_p,
            'seasonal_strength': seas,
            'trend_strength': trend,
            'missing_rate': np.mean(np.isnan(history)),
            'data_length': n,
            'recent_volatility': recent_vol,
            'period': period
        }

    @staticmethod
    def _auto_period(history):
        n = len(history)
        if n < 20: return 7
        try:
            acf_vals = acf(history, nlags=min(n//2,50))
            peaks = [i for i in range(2,len(acf_vals)-1) if acf_vals[i] > acf_vals[i-1] and acf_vals[i] > acf_vals[i+1] and acf_vals[i]>0.2]
            return peaks[0] if peaks else 7
        except:
            return 7

    @staticmethod
    def compute_dtw_distance(vec1, vec2):
        if len(vec1)==0 or len(vec2)==0: return float('inf')
        n1, n2 = len(vec1), len(vec2)
        idx1 = np.linspace(0, n1-1, min(n1,n2))
        idx2 = np.linspace(0, n2-1, min(n1,n2))
        v1 = np.interp(idx1, np.arange(n1), vec1)
        v2 = np.interp(idx2, np.arange(n2), vec2)
        return np.sqrt(np.mean((v1-v2)**2))

    @staticmethod
    def recommend_window(history: np.ndarray) -> dict:
        n = len(history)
        if n < 10:
            return {
                'min_train_size': max(3, n // 2),
                'reason': '数据极短，使用一半数据作为训练窗口',
                'period': 7,
                'data_length': n
            }
        profile = DataProfiler.profile(history)
        period = profile.get('period', 7)
        seasonal = profile.get('seasonal_strength', 0.0)
        trend = profile.get('trend_strength', 0.0)
        adf_p = profile.get('adf_pvalue', 0.5)

        if seasonal > 0.5 and period > 1:
            recommended = 2 * period
            reason = f"强季节性(强度={seasonal:.2f})，推荐至少2个完整周期({2*period}个点)"
        elif seasonal > 0.3 and period > 1:
            recommended = int(1.5 * period)
            reason = f"中等季节性(强度={seasonal:.2f})，推荐至少1.5个周期({int(1.5*period)}个点)"
        elif trend > 0.5:
            recommended = min(n, max(50, n // 3))
            reason = f"强趋势(强度={trend:.2f})，推荐较长窗口({recommended}个点)以捕捉趋势"
        elif adf_p < 0.05 and seasonal < 0.3 and trend < 0.3:
            recommended = max(30, int(n * 0.3))
            recommended = min(n, recommended)
            reason = f"平稳序列(ADF p={adf_p:.4f})，较短窗口({recommended}个点)即可"
        else:
            recommended = min(n, max(50, n // 3))
            reason = f"高波动或随机游走，推荐较长窗口({recommended}个点)以稳定估计"

        recommended = min(recommended, n - 1)
        recommended = max(recommended, 10)
        return {
            'min_train_size': recommended,
            'reason': reason,
            'period': period,
            'data_length': n
        }
```

  - **单技能模式（single）**：仅注册指定技能。
  - **组合模式（ensemble）**：注册一个封装所有技能的 `EnsembleSkill`。
  - **纯 LLM 模式（--no_skills）**：注册表为空，LLM 直接预测。

### 2. 滚动起源评估
- `RollingOriginEvaluator` 从 `min_train_size` 开始滑动窗口，每次预测下一个点（`horizon=1` 时）或未来第 N 个点（`horizon>1` 时）。
- 每一步构建 `TaskInstance`，调用 `agent.predict(task)` 获取预测值。

### 3. Agent 预测内部流程
a. **特征提取**  
   `DataProfiler.profile(history)` 提取丰富的统计特征：  
   ADF、季节性强度、趋势强度、局部斜率、变点检测（CUSUM-like）、周期（月频强制12）、频域特征（主导频率、频谱熵）、多尺度窗口快照等。

b. **技能匹配**  
   `SkillMatcher` 遍历所有技能，根据 `state_card` 进行硬过滤（`when_to_use` / `when_not_to_use`），保留符合条件的技能。  
   - 动态调整 `top_k`：根据季节性和趋势强度决定候选数量。  
   - 原型 DTW 排序：若有成功案例原型则计算相似度，否则给无原型技能一个基础相似度。  
   - 返回前 `top_k` 个候选。

c. **LLM 决策**  
   `LLMPlannerAgent._llm_branch_decision` 将特征报告（含滑动窗口统计、频域信息）和候选技能的状态卡发送给智谱 GLM-4，LLM 返回 JSON 指定 `best_skill`。  
   - 当某个技能连续使用超过 3 次时，Prompt 中会增加警告，鼓励尝试其他技能。

d. **技能执行**  
   调用选定技能的 `execute(history, horizon)` 进行数值预测。

e. **验证与回退**  
   - 计算 Z‑score 检查预测是否合理（3σ 内）。  
   - 若验证失败，使用候选列表中的下一个技能重试。  
   - 若所有候选均失败，回退到 `naive` 或历史均值。

### 4. 指标计算
- 收集所有预测值与真实值，计算：  
  RMSE、MAE、sMAPE、MASE、RMSSE、OWA、MdAPE（7项指标）。  
- 打印报告并保存详细结果到 CSV。

### 5. 日志记录
- 每次决策的 `profile`、`candidates`、`guidance`、`skill_predict` / `fallback` 等信息均写入独立的日志文件，便于调试和分析。

**核心设计原则**：LLM 仅负责“选择技能”，数值预测完全由本地技能算法完成；通过状态卡和分支决策实现数据驱动的自适应预测；新增频域特征和动态 top_k 增强了系统对复杂时序的感知能力。
